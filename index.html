Lists/Dynamic Array

Overview

Lists: In programming, a list is a data structure that stores a collection of elements. Lists are often used to store items of the same type, such as numbers, strings, or objects. Unlike arrays, lists can dynamically grow or shrink in size as elements are added or removed.
Dynamic Array: A dynamic array is a type of array data structure that automatically resizes itself to accommodate a varying number of elements. Unlike static arrays, which have a fixed size determined at compile time, dynamic arrays can expand or contract in size dynamically during program execution.
Now, when we combine "Lists" and "Dynamic Array" together, "Lists/Dynamic Array" likely refers to a data structure or programming concept that combines the characteristics of both lists and dynamic arrays. It suggests a data structure that behaves like a list, allowing for flexible storage of elements, and internally may be implemented using a dynamic array to manage its storage efficiently.
In many programming languages, lists or arrays with dynamic resizing capabilities are commonly used to implement data structures such as lists, vectors, or ArrayLists, providing the ability to store collections of elements that can grow or shrink as needed.
In summary, "Lists/Dynamic Array" likely refers to a data structure or concept that provides dynamic resizing capabilities similar to a dynamic array while offering list-like functionality for storing and manipulating collections of elements.

Methods of a list

These are common methods used with lists:

  • .Add(item): Adds an item to the end of the list.
  • .AddRange(collection): Adds the elements of a collection to the end of the list.
  • .Remove(item): Removes the first occurrence of a specific item from the list.
  • .RemoveAt(index): Removes the item at the specified index from the list.
  • .Contains(item): Determines whether the list contains a specific item.
  • .Find(predicate): Finds the first element that matches the conditions defined by the specified predicate.
  • .Count: Gets the number of elements contained in the list.
  • .Capacity: Gets or sets the total number of elements the list can hold without resizing.
  • .Clear(): Removes all elements from the list.
  • .FindIndex(predicate): Searches for an element that matches the conditions defined by the specified predicate and returns the zero-based index of the first occurrence within the entire list.
  • .ToArray(): Copies the elements of the list to a new array.
  • .Max(): Returns the maximum value in the list.
  • .Min(): Returns the minimum value in the list.
  • .Sum(): Computes the sum of the sequence of numeric values in the list.
  • .Sort(): Sorts the elements in the list.
  • .Reverse(): Reverses the order of the elements in the list.
  • .Insert(index, item): Inserts an item into the list at the specified index.

Linked-List

Overview

Linked Lists: In computer science, a linked list is a linear data structure consisting of a sequence of elements, called nodes, where each node contains a data element and a reference (or link) to the next node in the sequence. Unlike arrays, which store elements in contiguous memory locations, linked lists use pointers or references to connect nodes, allowing for dynamic allocation of memory and efficient insertion and deletion of elements. Linked lists come in various forms, including singly linked lists, doubly linked lists, and circular linked lists, each with different properties and characteristics.

Singly Linked Lists: In a singly linked list, each node contains a data element and a single reference to the next node in the sequence. The last node typically points to a null reference to signify the end of the list.

Doubly Linked Lists: In a doubly linked list, each node contains both a data element and references to both the next and previous nodes in the sequence. This allows for traversal in both forward and backward directions.

Circular Linked Lists: In a circular linked list, the last node points back to the first node, forming a circular structure. This can be useful for certain applications, such as representing a circular buffer or implementing algorithms that require cyclic behavior.

Linked lists offer several advantages and disadvantages compared to other data structures such as arrays. They excel at dynamic memory allocation, insertion, and deletion operations, particularly when elements need to be added or removed frequently from the middle of the list. However, they may have higher memory overhead due to the additional storage required for pointers, and accessing elements by index may require linear traversal, resulting in slower performance for random access.

Overall, linked lists are a fundamental data structure in computer science, widely used in various applications and algorithms, including memory management, graph traversal, and implementing other data structures like stacks, queues, and hash tables. Understanding linked lists and their properties is essential for any programmer or computer scientist.

Methods of a linked list

These are common methods used with linked lists:

  • .AddFirst(Value) - Adds a value as the first node in the linked list.
  • .AddLast(Value) - Adds a value as the last node in the linked list.
  • .AddAfter(index, Value) - Adds a value after a specified index in the linked list.
  • .Remove(Value) - Removes a node with the specified value from the linked list.
  • .Contains(Value) - Checks if the linked list contains a specific value.
  • .Find(Value) - Finds the first occurrence of a value in the linked list.
  • .First - Gets the first node in the linked list.
  • .Last - Gets the last node in the linked list.

How to implement your linked list (Singular)

This is how you can code a linked list from scratch:

                
    using System;
    
    public class Node
    {
        public int Value { get; set; }
        public Node next { get; internal set; }
    
        public Node(int val)
        {
            Value = val;
            next = null;
        }
    }
    
    public class MyLinkedList
    {
        public Node head;
    
        public MyLinkedList()
        {
            head = null;
        }
    
        // Other methods omitted for brevity...
    
        public void ReverseList()
        {
            Node current = head;
            Node previous = null;
    
            while (current != null)
            {
                Node tempcur = current.next;
                current.next = previous;
                previous = current;
                current = tempcur;
            }
            head = previous;
        }
    }
    
    class Program
    {
        static void Main(string[] args)
        {
            MyLinkedList list = new MyLinkedList();
            list.AddAtTail(1);
            list.AddAtTail(2);
            list.AddAtTail(3);
    
            // Display the original list
            Console.WriteLine("Original List:");
            Node temp = list.head;
            while (temp != null)
            {
                Console.Write(temp.Value + " ");
                temp = temp.next;
            }
            Console.WriteLine();
    
            // Reverse the list
            list.ReverseList();
    
            // Display the reversed list
            Console.WriteLine("Reversed List:");
            temp = list.head;
            while (temp != null)
            {
                Console.Write(temp.Value + " ");
                temp = temp.next;
            }
            Console.WriteLine();
        }
    }
