Regular Expressions

Introduction

Regular expressions (regex or regexp) are patterns used to match character combinations in strings. They are widely used in text processing and provide a powerful way to search, manipulate, and validate strings.

Wildcards

A wildcard in regex represents any character or set of characters. Here are some common wildcards:

  • .: Matches any single character except newline.
  • \d: Matches any digit (equivalent to [0-9]).
  • \D: Matches any non-digit character (equivalent to [^0-9]).
  • \w: Matches any word character (equivalent to [a-zA-Z0-9_]).
  • \W: Matches any non-word character (equivalent to [^a-zA-Z0-9_]).
  • \s: Matches any whitespace character (space, tab, newline).
  • \S: Matches any non-whitespace character.
  • \b: Matches a word boundary.
  • \B: Matches a non-word boundary.
  • \t: Matches a tab character.
  • \n: Matches a newline character.
  • \r: Matches a carriage return character.
  • \v: Matches a vertical tab character.
  • \f: Matches a form feed character.
  • \0: Matches a null character.
  • \xHH: Matches the character with the hex code HH.
  • \uHHHH: Matches the Unicode character with the hex code HHHH.
  • [abc]: Matches any single character in the specified set.
  • [^abc]: Matches any single character not in the specified set.
  • [a-z]: Matches any single character in the specified range.
  • [^a-z]: Matches any single character not in the specified range.
  • \Q...\E: Matches the enclosed characters literally, ignoring regex metacharacters.
  • |: Alternation, matches either the pattern before or after the pipe.
  • (...): Groups multiple tokens together.
  • (?:...): Non-capturing group, groups multiple tokens without capturing.
  • (?=...): Positive lookahead, asserts that the following pattern matches.
  • (?!...): Negative lookahead, asserts that the following pattern does not match.
  • (?<=...): Positive lookbehind, asserts that the preceding pattern matches.
  • (?: Negative lookbehind, asserts that the preceding pattern does not match.

Common Regex Strings

Here are some common regex patterns:

  • Email Validation: ^\w+@[a-zA-Z_]+?\.[a-zA-Z]{2,3}$
  • URL Matching: ^(http|https)://[a-zA-Z0-9\-.]+\.[a-zA-Z]{2,3}(/\S*)?$
  • IP Address: \b(?:\d{1,3}\.){3}\d{1,3}\b
  • HTML Tags: <\/?[\w\s="/.':;#-\/\?]+>
  • Date (MM/DD/YYYY): ^(0[1-9]|1[0-2])\/(0[1-9]|[12][0-9]|3[01])\/(19|20)\d{2}$
  • Time (HH:MM AM/PM): ^((1[0-2]|0?[1-9]):([0-5][0-9]) ([AaPp][Mm]))$
  • Hexadecimal Color Code: ^#([A-Fa-f0-9]{6}|[A-Fa-f0-9]{3})$

Using Regex in C#

In C#, you can use the System.Text.RegularExpressions namespace to work with regular expressions. Here's a basic example:


using System;
using System.Text.RegularExpressions;

class Program {
    static void Main() {
        string input = "Hello, World!";
        string pattern = "Hello, (.+?)!";
        Match match = Regex.Match(input, pattern);
        if (match.Success) {
            Console.WriteLine("Found match: " + match.Value);
        } else {
            Console.WriteLine("No match found.");
        }
    }
}
